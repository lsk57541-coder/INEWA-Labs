# 파트너 여정 감사 (온보딩 0단계 조사)

파트너(유튜버) 모집이 핵심 전략인데, 온보딩을 얹기 전에 "안내할 대상(기능·화면)이 매끄러운지"
먼저 확인하는 0단계 조사다. 파트너의 4단계 여정(가입·연동 → 영상 등록 → 장소 관리 →
대시보드)을 코드로 추적해 현재 상태와 불편/막힘 지점을 목록화했다. **발견은 전부 "제안"이며
실제 수정은 별도 지시에서 한다.** (이 문서 작성 시점 코드 수정 0)

---

## 1. 가입·연동

**현재 상태:** 메뉴 "유튜버 파트너" → [/partner/apply](../src/app/partner/apply/page.tsx)
(혜택 5개 + 연동 3단계 + Q&A 랜딩) → CTA "YouTube 채널 연동하기"(`/api/auth/youtube/start`)
→ 구글 OAuth → 콜백에서 `completePartnerSignup` **심사 없이 즉시 승인** → `/partner/dashboard`.
실제 신원 세션은 카톡, 구글은 채널 소유권 증명 + 토큰 저장. 로그인된 상태로 진입하면 매끄럽다.

**발견된 문제:**
- **[비로그인 상태에서 "유튜버 파트너" 클릭 시]** `/partner/apply`가 `if(!user) redirect('/login')`
  ([apply/page.tsx:116](../src/app/partner/apply/page.tsx))로 로그인 페이지로 튕기는데
  ① "파트너 신청하려면 로그인 먼저"라는 **맥락 안내가 없고** ② 로그인 후 apply로 **복귀 안 함**
  (홈으로) → 유저가 메뉴를 다시 열어 찾아가야 함. / **[우선순위: a]** /
  고치는 법: apply redirect에 `?next=/partner/apply` + 로그인 복귀 처리, 로그인 화면에 파트너
  맥락 배너(옵션). ※로그인 콜백 흐름을 건드리므로 인증 구조 미변경 제약과 겹침 — 수정 시 주의.
- **[카톡→구글 2단계가 안내 안 됨]** apply 랜딩은 "YouTube 채널 연동"만 말하고, 그 앞에 카톡
  로그인이 선행돼야 함을 설명 안 함. 로그아웃 진입 시 "카톡 → 유튜브" 2단계임이 불명확. /
  **[우선순위: b]** / 고치는 법: 온보딩/랜딩에 "카톡 1초 로그인 후 유튜브 연동" 1줄. (온보딩 재료)
- **[클릭 수]** 로그인된 비파트너: 메뉴(1)→유튜버 파트너(2)→연동하기(3)→구글 동의(4)→대시보드.
  3~4클릭, 막힘 없음(양호). 비로그인은 로그인 왕복이 추가됨.

## 2. 영상 등록

**현재 상태:** 대시보드 "영상으로 장소 등록하기" 또는 장소관리 →
[extract](../src/app/partner/dashboard/places/extract/page.tsx). [ExtractPlacesForm](../src/components/partner/ExtractPlacesForm.tsx):
탭(내 채널 영상목록 / URL 직접) → 영상 선택 → 미리보기("이미 N개 등록" 안내) → 지역 입력 →
"자동 추출" → 로딩("추출 중…" / "좌표 자동 채우는 중…") → 장소행 목록(이름/주소/카테고리/검색)
→ 일괄 저장 → 완료화면 "N개 신규 / M개 갱신".

**발견된 문제:**
- **[여러 영상 반복 번거로움]** 완료화면에 "장소 목록으로 돌아가기"만 있고 **"다른 영상 등록"
  바로가기 없음** → 여러 개 등록 시 목록↔extract 왕복. / **[우선순위: b]** /
  고치는 법: [완료화면](../src/components/partner/ExtractPlacesForm.tsx)에 "다른 영상 등록하기"
  링크 1개 추가.
- **[로딩/실패 안내]** 로딩 텍스트·에러 텍스트·추출 0개 안내("직접 추가") 모두 있음 — 양호(무반응
  구간 없음).
- **[추출 2단계 클릭]** 영상 선택 후 "자동 추출"을 또 눌러야 함. 지역 입력 후 추출 의도라 현행도
  합리적이나 통합 여지. / **[우선순위: c]**.
- **[N신규/M갱신 가시성]** 완료화면에 있으나 `text-xs` 회색이라 작음. 재등록=갱신 확인이 약함. /
  **[우선순위: c]**.

## 3. 장소 관리 ★핵심(온보딩 대상 = 데이터 저장·수정)

**현재 상태:** [places/page.tsx](../src/app/partner/dashboard/places/page.tsx)
탭(전체/지도표시중/검토중/반려됨/비공개) + [PlacesList](../src/app/partner/dashboard/places/PlacesList.tsx)
+ [PlaceRow](../src/app/partner/dashboard/places/PlaceRow.tsx). 인라인 편집(이름/주소/카테고리/URL,
onBlur 자동저장). AI 미검증 장소는 상단 정렬 + 파란 "맞아요/아니에요" 배너.
하단: 클릭수 · 비공개 처리 · 삭제.

**발견된 문제:**
- **[★모바일 편집 어포던스]** 편집 필드가 `border-transparent hover:border-gray-200`
  ([PlaceRow.tsx:98,110,119](../src/app/partner/dashboard/places/PlaceRow.tsx)) —
  **터치기기엔 hover가 없어 "편집 가능"임이 안 보임.** 그냥 텍스트처럼 보여 수정 가능한지 모르고,
  `text-xs`라 작음. (기존 "모바일에서 작아 안 보인다" 지적과 직결) / **[우선순위: a]** /
  고치는 법: 편집 필드에 상시 옅은 테두리/배경 + 글자 `text-sm` 상향 + 편집 힌트 아이콘.
- **[source='ai'만 검증버튼]** `isAiUnverified = source==='ai' && unverified`
  ([PlaceRow.tsx:89](../src/app/partner/dashboard/places/PlaceRow.tsx))라 list/coords/null 소스엔
  검증 UI 없음 → 유튜버가 "왜 어떤 건 맞아요/아니에요가 있고 어떤 건 없지?" 헷갈림. /
  **[우선순위: b]** / 고치는 법(문서·온보딩): "AI가 자동으로 찾은 장소만 확인 요청" 설명, 또는
  비-AI 장소에도 짧은 상태 라벨.
- **[삭제/비공개 후 행방 불명]** 삭제·비공개·reject 모두 낙관적으로 목록에서 사라짐(`onHidden`)인데
  "어디로 갔는지"(비공개 탭엔 있고 삭제는 목록 밖) 피드백 없음 → 방금 처리한 게 헷갈림. /
  **[우선순위: b]** / 고치는 법: 액션 후 토스트("비공개로 이동했어요 / 삭제했어요") 또는 탭 유도.
- **[인라인 저장 성공 피드백 없음]** onBlur 자동저장인데 성공 표시가 없음(에러만 표시) → 저장됐는지
  모름. / **[우선순위: c]** / 고치는 법: 저장 후 짧은 "저장됨" 표시.
- **[검증 번복]** confirmed↔rejected 상호변경("맞아요로 변경/아니에요로 변경") 동작 명확 — 양호.

## 4. 대시보드

**현재 상태:** [dashboard/page.tsx](../src/app/partner/dashboard/page.tsx) — 채널연동 카드(구독자수·
연동됨✓ / 미연동 시 "재연동") + CTA(영상으로 등록) + 통계 2칸(이번달 장소수 / 클릭수, 전월 대비)
+ 네비카드(장소관리 / 설정).

**발견된 문제:**
- **[빈 상태 — 장소 0개]** "아직 등록된 장소가 없어요, 영상으로 등록해보세요 →" 친절한 안내 있음.
  화면 안 깨짐 — 양호.
- **[클릭수 0 비대칭]** 클릭수 칸은 빈 상태 안내 없이 "0" + "전월 대비 0%" 표시 → 장소 칸과 비대칭,
  0인데 통계처럼 보여 어색. / **[우선순위: c]** / 고치는 법: 클릭 0일 때 "아직 클릭 없음 · 지도에
  노출되면 쌓여요" 안내.
- **[changeRate 오해]** previous=0 & current>0이면 '+100%'로 표시
  ([page.tsx:5](../src/app/partner/dashboard/page.tsx)) → 신규인데 +100%가 오해 소지. /
  **[우선순위: c]** / 고치는 법: previous=0 & current>0이면 '신규'/'NEW'.

## 우선순위 요약표

- **(a) 모집 전 꼭:**
  - 장소관리 **모바일 편집 어포던스/가독성**(3) — 온보딩 대상 화면이 정작 모바일에서 수정 가능한지
    안 보이면 온보딩 무의미.
  - 비로그인 "유튜버 파트너" 클릭 시 **맥락 안내 + 로그인 후 복귀**(1) — 첫 진입 이탈 지점.
- **(b) 있으면 좋음:**
  - 완료화면 "다른 영상 등록" 링크(2)
  - 삭제/비공개 후 **토스트 피드백**(3)
  - source='ai' 검증 조건 **온보딩 안내**(3)
  - 카톡→구글 2단계 안내(1)
- **(c) 나중에:**
  - 추출 2단계 클릭 통합(2) / N신규·M갱신 가시성(2) / 인라인 저장 성공 표시(3) /
    클릭수 0 비대칭(4) / changeRate 신규 처리(4)

## 온보딩 재료 (유튜버가 헷갈릴 만한 지점 = 나중에 온보딩 목차)

- **1단계 가입:** "파트너는 카톡 로그인 → 유튜브 연동 2단계. 심사 없이 연동 즉시 활성화."
- **2단계 등록:** "영상 선택 → 자동 추출 → 저장. 같은 영상을 다시 등록해도 새로 안 생기고 기존
  장소가 갱신돼요."
- **3단계 관리:** "목록에서 바로 수정(탭으로 상태 구분). AI가 자동으로 찾은 장소만 '맞아요/
  아니에요'로 확인. 비공개=일시 숨김, 삭제=제거(복구는 문의)."
- **4단계 대시보드:** "장소가 지도에 노출되면 클릭수가 쌓여요. 통계는 월 단위."
