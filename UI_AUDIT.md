# UI/UX 감사

## 범위 및 방법론

- 평가 기준: 일관성 / 타이포그래피 위계 / 색상 의미 / 모바일 대응 / 빈상태·에러상태 / 지도 중심 클릭효율
- 우선순위: **상** = 핵심 플로우 마찰, **중** = 눈에 띄지만 차단 아님, **하** = 다듬기
- 각 항목에 `[코드 확인됨]` 또는 `[스크린샷 필요]` 표시 — 코드만으론 실제 체감(겹침, 대비, 터치감)을 확정할 수 없는 항목은 후자로 표시

## 1. 메인 지도 화면 (`/`, `SearchMap.tsx`)

가장 우선순위가 높은 화면 — 사용자가 서비스에 들어와서 첫인상을 받는 곳.

### 검색 패널
- ✅ **완료** 지도-중심 정보 밀도 — 검색 전부터 풀옵션 폼이 화면 절반을 차지하던 구조를 구글맵/카카오맵 방식으로 재설계: 기본은 한 줄짜리 검색바만 보이고(`optionsOpen=false`), 바를 탭하면 바로 입력 가능, `▾` 아이콘을 눌러야 모드탭/주소입력/반경/투명도/검색버튼이 펼쳐짐(`max-height` 애니메이션). "검색창 열기" 알약 버튼과 ✕ 완전히 숨기기 기능은 제거(참조 앱들도 검색바를 완전히 숨기는 기능이 없음). 검색 실행 후엔 옵션만 접히고 입력했던 검색어는 바에 계속 보임.
- ✅ **완료** 위치 미설정 시 "위치를 먼저 설정해주세요" 에러로 막던 문제 — 키워드만 입력하고 검색하면 위치 설정 화면을 찾아가야 했음(특히 바가 접혀있는 새 구조에서 더 자주 노출될 흐름). 위치가 없으면 검색 시점에 GPS를 자동으로 한 번 요청해서 바로 검색하도록 변경 (`requestCurrentPosition()`), 실패 시에만 에러 표시.
- ✅ **완료** 액센트 컬러가 "선택 상태"와 "액션 버튼"에 동시에 쓰이던 문제 — 키워드/채널 탭, 반경 칩, 정렬 칩의 선택 상태를 `bg-accent`(블루, 선택 상태 전용 토큰)로 통일하고, 주소입력 옆 소형 검색 버튼은 `bg-gray-100`(보조 액션)으로 교체해 액션 버튼과 분리함.
- **[정정]** ~~"현재 위치로" 버튼이 폼 안에 묻혀 발견성이 낮음~~ — 코드 확인 결과 지도 위에 별도 플로팅 위치 버튼(크로스헤어 아이콘, `line 845`)이 이미 있었음. 폼 안의 텍스트 링크는 수동 주소 입력 중에도 실제 GPS를 강제로 다시 잡는 별도 용도라 중복이 아님. **실제 문제 아님으로 종료.**
- ✅ **완료** 탭 토글(키워드/채널)과 반경/정렬 칩이 `rounded-lg`라 즐겨찾기 오버레이 탭(`rounded-full`)과 불일치하던 문제 — 전부 `rounded-full`로 통일.
- 좋은 점: 투명도 슬라이더로 패널 너머 지도를 미리 볼 수 있게 한 디테일, 검색 후 옵션이 접히는 동작은 유지.

### 지도 / 마커
- ✅ **완료** 마커 클러스터링 가독성 — 원인 확인: `groupByLocation`이 줌 레벨과 무관하게 고정 80m 반경으로만 묶어서, 반경을 넓게(예: 10km) 검색할수록 화면상으로는 같은 80m가 훨씬 가까운 픽셀 거리가 되어 핀이 겹쳐 보였음. 검색 반경에 비례해 클러스터링 거리를 늘리는 `clusterThresholdKm()`을 추가해 해결 (`SearchMap.tsx`).
- ✅ **완료** 결과 카드 액션 아이콘들의 색 의미가 충돌하던 문제 — 찜(♥)과 신고(🚩)가 둘 다 `red-500`을 써서 색만으론 구분이 안 됐음. 찜은 `text-favorite`(금색, 지도 마커의 찜 표시와도 통일), 신고는 `text-danger`(레드)로 분리.
- ✅ **완료** 결과 필터 탭(롱폼/쇼츠)이 아이콘만 있고 텍스트 라벨이 없던 문제 — 모바일에선 `title` 툴팁이 안 보여 실질적으로 라벨이 없는 상태였음. 아이콘 옆에 "롱폼"/"쇼츠" 텍스트 추가.

### 결과 바텀시트 / 마커그룹 상세시트
- ✅ **완료** 빈 상태가 회색 텍스트만 있고 다음 행동 제안이 없음 — 실제로는 두 경우가 있었음: ①검색 자체가 0건이면 검색바 옆에 에러 텍스트만 뜸(그대로 둠, 리스트 영역 자체가 안 열림), ②필터(롱폼/쇼츠) 적용 후 0건이 되면 리스트 영역이 아무 표시 없이 완전히 비어있었음(진짜 빈 상태). 후자에 "이 필터에 맞는 영상이 없어요" + 전체 보기로 전환 버튼 추가.
- **[종료]** 결과 리스트 썸네일(`w-14 h-8`)이 즐겨찾기 오버레이 썸네일(`w-20 h-12`)보다 작은 것 — 검색 결과 리스트는 한 화면에 여러 줄을 보여줘야 하고 즐겨찾기는 상대적으로 적은 항목을 더 자세히 보여주는 화면이라, 의도된 정보밀도 차이로 판단해 변경하지 않음.
- 좋은 점: 마커 1개를 클릭했을 때는 한 줄 헤더("이 위치의 영상 1개 · 조회수순") + 카드 1개로 깔끔하게 축소됨 — 검색 직후 패널과 달리 이 상태에서는 오버레이 겹침 문제가 없음.

### 영상 미리보기 모달 / 신고 모달
- **[정정]** ~~신고 제출, 즐겨찾기 토글 등 액션 후 성공 토스트가 없음~~ — 코드를 다시 보니 대부분 이미 즉시 피드백이 있었음: 찜/가본곳은 아이콘이 그 자리에서 바로 채워짐, 신고 제출은 모달 안에 "신고가 접수되었습니다" 텍스트가 뜨고 1.2초 후 자동으로 닫힘. 별도 토스트 시스템을 새로 만드는 건 이미 있는 피드백에 비해 과한 추가로 판단해 보류. **실제 문제는 아니었음으로 종료.**
- ✅ **완료** 버튼 클릭 시 로딩 스피너가 없음 — `disabled` + 텍스트 변경("처리 중…")만 있던 검색 버튼/신고 제출 버튼에 작은 회전 스피너 추가(`Spinner` 컴포넌트).
- ✅ **완료** 영상 모달이 떠 있을 때 모달 바로 아래 같은 영상의 카드가 다시 한 번 보여 정보가 중복되던 문제 — 모달(`selectedVideo`)이 열려 있을 땐 마커그룹 시트를 함께 렌더링하지 않도록 조건 추가.

### 즐겨찾기 화면 (`관심목록`)
